Tutorial 2

At the end of this session you will be able to,

  • Understand what version control is
  • Learn how to add your project to github
  • Learn how to edit your code and save the changes to github
  • Learn how to create a project in Android Studio



Fig 1: Version Control Workflow

Version control helps a team to collaborate and work together without any hassles. For easier understanding of version control workflow, we will use Git. Git helps in developing a project without any repetitions of a work by various team members and in keeping every member of team up to date.

The above sketch describes the overview of the workflow and some of the popular terms used during version control.

    Now, let’s create an android project.
  • Open Android Studio in your PC. Once you do this, a window as shown below will appear.
  • Since you are creating a new project from scratch, you will click on “Start a new Android Studio Project”. You will be prompted to another screen where you’ll be asked to give the project name and company domain (a unique name to identify as yours, you can just give your name here) as seen in the screenshot below.

After you have given the name of your application, you’ll be given the option to choose the target devices.

In this tutorial you will learn to develop a simple mobile application, so the targeted device you’ll choose will be “Phone and Tablet” as shown in the screenshot above.

Now, you’ll be asked to choose an activity type for your application. The activity type is nothing but a screen form for your application. Based on the application this activity form changes. For this tutorial, you can select the “empty activity” form as shown in the screenshot.

Next, you’ll be asked to give the above selected activity a name and a layout name. Android Studio already gives a suggestion for the names, you can just use those.

Once you click on “Next”, you’ll get a screen like this:

This screen shows that the project has been successfully created. Now click on “Finish”.

The above window pops up. This window says that the entire setup required for the project is being built.

Once the built is done, you’ll see a screen like above. Both a jar file and an XML file are created for the activity.

Now, you will push (When you update the existing project in the repo with the changes you made locally, that is, you are trying to make some changes to the project.) this project into your earlier created Git account.

First you are to sign in to your Git account.

Now, click on “New Repository” and give a name to the repo (The database which contains all the files created).

After creating the repo, you’ll see a screen like the one shown below.

The project is on your Git account. Now, you will be adding your team members as contributors for the project.

In the above shown window, click on “Settings” and then on “contributors” present in the left corner.

You can give the names of your team members here and once they accept your invite to collaborate, you will have a shared repo among each other.

Each team member after accepting the invite can clone the project onto their local system. The option is available in green color on the right side in the below shown screenshot.

Once the team member has his/her copy on their local system, he/she can just open the project and make any kind of changes.

And when the team member is sure of the changes he/she made, the member can merge (Incorporating the changes made by you on your local file to the repo) their code into a branch other than the “master branch” (created by the team leader).

Before merging the code, one can make a “pull (When you get the last updated version of the project along with the changes you made locally) request”, by clicking on “New Pull Request” seen on the below screenshot.

To check the differences between the branches before merging them, we go a head with pull requests. The screen looks like this:

Now, a team member can compare the changes and see the differences and then merge their code.
